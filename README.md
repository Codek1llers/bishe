# 3D 城市立体道路导航仿真系统

一个面向毕设演示与立体道路实验的 Web 系统，核心目标不是做通用地图，而是突出 **高架、匝道、桥下道路在 2D 与 3D 表达中的差异**，支持路径规划、导航仿真、模板场景切换与 JSON 自定义导入。

## 项目定位

本系统聚焦以下问题：

- 传统 2D 俯视地图难以清晰表达桥上、桥下、匝道之间的空间层级关系
- 立体道路场景在教学、汇报和算法演示中缺少易操作、可快速切换的实验平台
- 单纯展示型项目无法完成“切场景、选路线、算路径、看仿真、做对比”这一整套闭环

因此，本项目构建了一个 **模板优先、立体表达优先、演示闭环优先** 的导航仿真系统。

---

## 当前主要功能

### 1. 立体道路导航仿真
- 支持起点、终点选择
- 基于路径规划算法计算最优路线
- 支持车辆沿路线自动运动
- 支持暂停、继续、逐节点跳转、加减速

### 2. 2D / 3D 对比演示
- 可在 2D 和 3D 视图间切换
- 可切换跟车视角
- 适合展示高架、桥下、匝道在不同视角下的差异

### 3. 推荐路线与历史记录
- 提供高频体验路线
- 提供立体道路对比演示路线
- 自动记录最近使用路线，便于答辩时重复演示

### 4. 模板场景切换
- 当前校园 + T 型立交主场景
- 高架主路 + 桥下辅路
- 菱形互通
- 桥梁跨越 + 桥下通道

这些模板不是简单平面路口，而是带有：
- 顺滑弯道
- 匝道起伏
- 高架与桥下重叠投影
- 不同节点与不同路线结构

### 5. JSON 场景导入 / 导出
- 支持导出当前场景为 JSON
- 支持从 0 导入完整场景
- 适用于实验复现、场景备份和自定义扩展

### 6. 拥堵调节与路径反馈
- 支持道路拥堵设置
- 支持路径统计、道路构成分析、平均拥堵展示
- 支持根据路径结果生成智能建议

## 当前页面结构

### 总览
用于快速展示系统价值与整体道路统计：
- 道路总量
- 平均拥堵
- 立体道路数量
- 拥堵分布
- 深度用户价值说明

### 演示驾驶
系统核心页面，完成完整演示闭环：
- 选点
- 推荐路线
- 开始导航
- 2D/3D 切换
- 跟车视角
- 拥堵实验
- 导航提示与路径信息查看

### 场景搭建
强调“模板切换 + JSON 导入”，而不是逐条造路：
- 一键切换不同立体道路模板
- 粘贴 JSON 快速生成新场景
- 导出当前实验场景

---

## 为什么这个系统有竞争力

相比纯展示型竞品，本系统更适合毕设、实验和教学，因为它具备：

1. **完整闭环**：不是只看地图，而是从场景切换到路线演示一步完成
2. **立体表达优势明确**：桥上、桥下、匝道切换比 2D 更清晰
3. **模板优先降低门槛**：用户不需要手工逐段构路
4. **JSON 保留扩展能力**：既能快速演示，也能完全自定义
5. **适合答辩讲述**：每个模板都可直接作为实验案例展示

## 推荐演示流程

### 演示流程 1：基础导航
1. 进入 `演示驾驶`
2. 选择起点和终点
3. 点击 `开始导航`
4. 播放车辆动画

### 演示流程 2：立体道路对比
1. 选择一条包含上桥 / 下桥的推荐路线
2. 先使用 2D 视图观察
3. 再切换到 3D 视图
4. 开启跟车视角观察匝道与高架层级变化

### 演示流程 3：切换模板
1. 进入 `场景搭建`
2. 点击任一模板场景
3. 返回 `演示驾驶`
4. 重新选择路线并开始仿真

### 演示流程 4：JSON 自定义场景
1. 将完整场景 JSON 粘贴到输入区
2. 点击导入
3. 回到演示页验证路径与仿真效果

## 项目后续可继续增强的方向

如果继续作为毕设深化，建议优先增加：

1. 模板缩略图预览
2. 模板推荐起终点自动填充
3. 多路线并行对比
4. 场景快照与实验记录
5. 导航过程日志导出
6. 不同算法路径结果对比（如 Dijkstra / A*）

---

## 适合答辩强调的创新点

- 从“平面导航”转向“立体道路空间导航表达”
- 通过 2D / 3D 对比突出系统必要性
- 模板化场景使实验切换成本大幅降低
- 支持从固定模板演示到 JSON 自定义扩展
- 将可视化、路径规划、导航仿真整合为一个可交互系统
